1 Introduction

A big name for a small program. I got sick of creating paths in inkscape so I wrote this. The great advantage is that if you get it wrong (you will) it only takes a moment to edit the text file, run the program and see the results. The only paths produced are line segments (hence simplePathWriter), no fancy splines or anything.

2 Synopsis

simplePathWriter [-h] [-v] [-s [-a]] [-p] [-o outputFile] [file]
produces svg path from up/down/left/right/move
instructions, combined with pen up/down commands
defaults to writing path data to stdout

options:
usage: simplePathWriter
options are:
    -a          : adjust. Ensure path is inside svg first quadrant.
    -h          : help. Writes usage text and exits.
    -o          : output file. Write result to output.
    -p          : path. Write complete path to stdout.
    -s          : svg. Write complete svg to stdout.
    -v          : version. Writes version and exits.

3 Options

  • -o file write to file, not stdout

  • -p write the complete path, with black stroke and red fill, like this:

    <path stroke="black"
stroke-width="1"
fill="red"
class="pathWriter"
d="h-20 m10 0 m0 10 v-20 "/>

    otherwise the output is like this:

    h-20 m10 0 m0 10 v-20

  • -s write complete svg containing the path.

  • -a if -s is specified, to save the bother of ensuring you’re always in the first quadrant (and bear in mind svg’s y direction is down) this will rejig the numbers so the path is on the page.

4 Commands

commands are

  • pu

    pen up

  • pd

    pen down

  • m integer1 integer2

    move right integer1 pixels and up integer2 pixels

  • u integer

    up integer pixels

  • d integer

    down integer pixels

  • l integer

    left integer pixels

  • r integer

    right integer pixels

All moves are relative. White space is optional. A negative integer moves in the opposite sense to that specified, eg. l 20 is the same as r -20. This is only really useful if you want to move left and/or down, eg. m -10 -30 moves left 10 and down 30.

5 Example

The file cross.simplepathwriter

pd
l 20
pu
r 10
d 10
pd
u 20

with this command line

..\source\simplePathWriter -s -a cross.simplePathWriter > cross.svg

gives

<svg xmlns="http://www.w3.org/2000/svg" width="20" height="20">
<path stroke="black"
stroke-width="1"
fill="red"
class="pathWriter"
d="m0 0 m20 10 h-20 m10 0 m0 10 v-20 "/>
</svg>


The file arrow.simplepathwriter

r 20 pd d 200 l 25
 m 35,-100 
 m 35,100 l25 u200
l 20

with this command line

..\source\simplePathWriter -s -a arrow.simplePathWriter > arrow.svg

gives arrow.svg

<svg xmlns="http://www.w3.org/2000/svg" width="70" height="300">
<path stroke="black"
stroke-width="1"
fill="red"
class="pathWriter"
d="m0 0 m5 0 m20 0 v200 h-25 l35 100 l35 -100 h-25 v-200 h-20 "/>
</svg>


6 Grammar

.simplepathwriter files conform to this grammar:

simplePathWriter :
  commandList END_OF_FILE
| ε
;
commandList :
  commandList command
| command
;
length :
  INTEGER
;
command :
  PEN_UP
| PEN_DOWN
| MOVE length COMMA length
| UP length
| DOWN length
| LEFT length
| RIGHT length
;
